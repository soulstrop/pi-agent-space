# The Rosetta Stone

Welcome to the bridge between the mathematical precursors and the Python implementation of **pi-agent-space**. 

Depending on your background and how you prefer to learn, we have provided two different paths through the project's core abstractions. Choose the one that matches your "entry point" into the system:

### [Path A: The Architect (Deductive)](ROSETTA_STONE_DEDUCTIVE.md)
**Direction:** Math (The Shape) $\to$ Haskell (The Verification) $\to$ Python (The Truth)
**Best if:** You enjoy starting with first principles, formal methods, or Category Theory. You want to understand the "Why" and the "Rules" before looking at the implementation details.

### [Path B: The Builder (Inductive)](ROSETTA_STONE_INDUCTIVE.md)
**Direction:** Python (The Truth) $\to$ Haskell (The Verification) $\to$ Math (The Shape)
**Best if:** You are primarily a Python developer. You prefer to see concrete code first and then see how that code can be "scaled up" into more powerful, verified abstractions.

---

## Why Two Paths?

This project is built on the conjecture that LLM-age software engineering benefits from "shifting left" into higher levels of abstraction. 
- The **Math** gives us a common language that is legible to both humans and LLMs.
- The **Haskell** gives us a way to prove that our "Math ideas" are structurally sound.
- The **Python** gives us the performance and ecosystem needed to run real-world trials.

By providing both paths, we hope to make the correspondence between these layers clear, regardless of which layer you are currently working in.
